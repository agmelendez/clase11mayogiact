Sesión 9: Regresión Logística Binaria aplicada a Gestión Marina Costera

Laboratorio autoguiado para estimar la probabilidad de ocurrencia de un evento: adopción de buenas prácticas, cumplimiento de normas, presencia de conflicto, participación comunitaria o riesgo socioambiental.

Propósito: pasar de modelos lineales para variables continuas a modelos probabilísticos para variables binarias, con énfasis en interpretación, clasificación y toma de decisiones.

Teoría 1 ¿Cuándo usar regresión logística?

Use regresión logística cuando la variable dependiente tiene dos categorías: sí/no, cumple/no cumple, participa/no participa, presencia/ausencia. El modelo estima la probabilidad de que ocurra el evento definido como Y = 1.

  • ¿Qué factores aumentan la probabilidad de que una comunidad participe en un programa de manejo costero?
  • ¿Qué variables predicen el cumplimiento de una norma ambiental?
  • ¿Qué condiciones están asociadas con presencia de conflicto socioambiental?
Error frecuente: no interprete los coeficientes logísticos como cambios directos en unidades de Y. Deben leerse como cambios en log-odds u odds ratios.

Teoría 2 Ecuación del modelo logit

La regresión logística transforma una combinación lineal de predictores en una probabilidad entre 0 y 1.

logit(p) = ln(p/(1-p)) = β₀ + β₁X₁ + β₂X₂ + ... + βₖXₖ

Luego, la probabilidad estimada se obtiene como:

p = 1 / (1 + exp(-η))
ElementoInterpretación
β positivoAumenta los odds del evento.
β negativoReduce los odds del evento.
exp(β)Odds ratio. Si OR = 1.5, los odds aumentan 50% por unidad de X, manteniendo las demás constantes.

Teoría 3 Diagnóstico y calidad del modelo

En logística no se evalúan normalidad ni homocedasticidad de residuales como en OLS. El foco cambia hacia especificación, convergencia, separación, multicolinealidad, calibración y desempeño de clasificación.

AspectoQué revisarHerramienta
DesbalanceFrecuencia de evento/no eventoTabla de frecuencias
ColinealidadPredictores redundantesVIF/correlaciones
ClasificaciónAciertos y errores por claseMatriz de confusión, sensibilidad, especificidad, F1
DiscriminaciónSeparación entre eventos y no eventosROC-AUC
CalibraciónProbabilidades bien alineadas con frecuencias observadasCurva de calibración, Brier score

Teoría 4 Interpretación para decisiones

La decisión operativa no depende solo del p-valor. También importa el umbral de clasificación. Si un falso negativo es costoso, puede bajarse el umbral para aumentar sensibilidad; si un falso positivo es costoso, puede subirse.

Lectura sustantiva: “controlando por experiencia, distancia y capacitación, la capacitación tiene OR = 2.1; es decir, los odds de adopción son aproximadamente 2.1 veces mayores en personas capacitadas que en no capacitadas”.

Módulo 1 Generación de datos sintéticos

Ejecute este bloque en Google Colab para crear un caso costero con evento binario.

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import statsmodels.api as sm
import statsmodels.formula.api as smf
from sklearn.metrics import confusion_matrix, classification_report, roc_auc_score, RocCurveDisplay
from statsmodels.stats.outliers_influence import variance_inflation_factor

np.random.seed(42)
n = 180

df = pd.DataFrame({
    "capacitacion": np.random.binomial(1, 0.48, n),
    "experiencia_anios": np.random.uniform(1, 35, n),
    "distancia_km": np.random.normal(18, 7, n),
    "indice_presion_turistica": np.random.normal(55, 15, n),
    "participacion_comunitaria": np.random.choice([1,2,3,4,5], n, p=[.10,.20,.30,.25,.15])
})

eta = (-1.20 + 1.05*df["capacitacion"] + 0.04*df["experiencia_anios"]
       - 0.05*df["distancia_km"] - 0.015*df["indice_presion_turistica"]
       + 0.38*df["participacion_comunitaria"])
df["prob_adopcion"] = 1/(1+np.exp(-eta))
df["adopta_buena_practica"] = np.random.binomial(1, df["prob_adopcion"])
print(df.head())
print(df["adopta_buena_practica"].value_counts(normalize=True))

Módulo 2 Ajuste del modelo logístico

formula = "adopta_buena_practica ~ capacitacion + experiencia_anios + distancia_km + indice_presion_turistica + participacion_comunitaria"
modelo = smf.logit(formula, data=df).fit(maxiter=200)
print(modelo.summary())

conf = modelo.conf_int()
or_table = pd.DataFrame({
    "OR": np.exp(modelo.params),
    "IC_2.5%": np.exp(conf[0]),
    "IC_97.5%": np.exp(conf[1]),
    "p_value": modelo.pvalues
})
print(or_table)

Módulo 3 Desempeño predictivo

df["p_hat"] = modelo.predict(df)
umbral = 0.50
df["pred_50"] = (df["p_hat"] >= umbral).astype(int)
print(confusion_matrix(df["adopta_buena_practica"], df["pred_50"]))
print(classification_report(df["adopta_buena_practica"], df["pred_50"], zero_division=0))
print("ROC-AUC:", roc_auc_score(df["adopta_buena_practica"], df["p_hat"]))
RocCurveDisplay.from_predictions(df["adopta_buena_practica"], df["p_hat"])
plt.show()

Módulo 4 Umbrales, VIF e interacción

for t in [0.30, 0.40, 0.50, 0.60, 0.70]:
    pred = (df["p_hat"] >= t).astype(int)
    print("\nUmbral", t)
    print(confusion_matrix(df["adopta_buena_practica"], pred))

cols_x = ["capacitacion", "experiencia_anios", "distancia_km", "indice_presion_turistica", "participacion_comunitaria"]
X = sm.add_constant(df[cols_x])
vif = pd.DataFrame({"variable": X.columns, "VIF": [variance_inflation_factor(X.values, i) for i in range(X.shape[1])]})
print(vif)

modelo_inter = smf.logit("adopta_buena_practica ~ capacitacion * participacion_comunitaria + experiencia_anios + distancia_km + indice_presion_turistica", data=df).fit(maxiter=200)
print(modelo_inter.summary())
