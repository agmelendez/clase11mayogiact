1. Propósito analítico

En gestión marina costera se observan sistemas complejos con variables biofísicas, socioeconómicas, institucionales y territoriales que suelen estar correlacionadas: presión turística, esfuerzo pesquero, calidad de agua, gobernanza local, conflictividad, infraestructura, cumplimiento normativo, percepción comunitaria y biodiversidad. El PCA permite reducir esa complejidad en dimensiones sintéticas interpretables; el análisis de clúster permite construir tipologías de territorios, comunidades, playas, sitios de monitoreo o unidades pesqueras con perfiles similares.

La combinación PCA + clúster no debe entenderse como una receta automática. Es un procedimiento exploratorio multivariado para ordenar información, detectar patrones y apoyar decisiones. Su valor depende de una buena definición conceptual de las variables, de la calidad de los datos y de una interpretación territorialmente informada.

2. Reglas metodológicas mínimas

  1. Unidad de análisis explícita: defina si cada fila representa un sitio, comunidad, playa, distrito, embarcación, hogar, organización o actor institucional. No mezcle niveles de análisis sin justificación.
  2. Variables conceptualmente defendibles: cada indicador debe representar una dimensión relevante del sistema costero. Evite incluir variables solo porque están disponibles.
  3. Dirección de los indicadores: documente si valores altos significan mayor presión, mejor condición, mayor capacidad o mayor vulnerabilidad. Invierta escalas cuando sea necesario para que la interpretación sea coherente.
  4. Variables numéricas comparables: convierta escalas ordinales con cautela y declare el supuesto. Por ejemplo, una escala Likert de 1 a 5 puede usarse como aproximación continua solo si se interpreta como índice ordinal agregado.
  5. Estandarización obligatoria: use z-score cuando las variables tienen unidades distintas o rangos muy diferentes. Sin estandarización, una variable con mayor varianza puede dominar artificialmente el PCA y el clúster.
  6. Revisión de correlaciones: PCA es más útil cuando existe estructura de correlación. Si las variables son prácticamente independientes, la reducción dimensional pierde sentido.
  7. Selección de componentes: combine varianza acumulada, scree plot, criterio de Kaiser cuando aplique, interpretabilidad de cargas y consistencia sustantiva.
  8. Validación de clúster: compare soluciones con codo, silhouette, estabilidad de centroides y sentido conceptual. El número de grupos no debe decidirse solo por un indicador automático.
  9. Lectura territorial: los grupos estadísticos no son automáticamente categorías sociales, ecológicas o administrativas reales. Valide los perfiles con conocimiento local, cartografía, entrevistas o revisión institucional.

Ruta de trabajo

Producto 1: matriz depurada, documentada y estandarizada.
Producto 2: PCA con cargas, varianza explicada, scores e interpretación de componentes.
Producto 3: clústeres comparados, interpretados, etiquetados y validados.
Advertencia: una solución estadística puede ser elegante pero irrelevante si no responde a una pregunta de manejo. Primero defina el problema; después aplique la técnica.
Error común: llamar “vulnerabilidad”, “resiliencia” o “gobernanza” a un componente solo porque una variable aparece con carga alta. El nombre debe surgir del conjunto de cargas, del signo, de la teoría y del contexto.

3. Supuestos y verificaciones antes del PCA

Antes de ejecutar PCA, el estudiante debe revisar si la matriz de datos tiene condiciones mínimas para una reducción dimensional razonable. Estas verificaciones no son trámites formales: permiten detectar variables mal medidas, indicadores redundantes, escalas invertidas, casos atípicos y problemas de interpretación.

Lectura metodológica: PCA no “descubre la verdad” del sistema costero. Construye ejes matemáticos que maximizan varianza. La interpretación depende de las cargas factoriales, del signo de las variables y de la pregunta de investigación.

4. Desarrollo metodológico del PCA

El PCA transforma un conjunto de variables originales posiblemente correlacionadas en nuevas variables no correlacionadas llamadas componentes principales. Cada componente es una combinación lineal de las variables estandarizadas. El primer componente explica la mayor varianza posible; el segundo explica la mayor varianza restante, y así sucesivamente.

En términos aplicados, si varias variables de presión humana se mueven juntas —por ejemplo, turismo, infraestructura, residuos y conflictividad— el PCA puede sintetizarlas en una dimensión de “presión antrópica”. Si variables como gobernanza, cumplimiento normativo y participación comunitaria se asocian positivamente, pueden formar una dimensión de “capacidad de gestión”.

4.1 Ecuación conceptual

PC1 = a1*z(variable_1) + a2*z(variable_2) + ... + ap*z(variable_p)

Donde z(variable) representa la variable estandarizada y a representa el peso o carga asociada a cada indicador. Las cargas permiten interpretar qué variables definen cada componente.

4.2 Cómo interpretar las cargas

SituaciónInterpretaciónPrecaución
Cargas altas y positivas en presión turística, esfuerzo pesquero y degradación.El componente puede representar presión antrópica o intensidad de uso.Verifique si calidad de agua aparece con signo contrario; eso refuerza la lectura.
Cargas altas en gobernanza, cumplimiento e infraestructura.El componente puede representar capacidad institucional o capacidad de gestión.No confundir infraestructura con buena gobernanza si no hay soporte conceptual.
Cargas mezcladas sin patrón claro.El componente puede ser difícil de interpretar.No fuerce nombres. Puede retenerse estadísticamente pero reportarse con cautela.
Una sola variable domina el componente.Podría no haber reducción real; el componente replica una variable original.Revise si conviene mantener la variable original en vez del componente.

5. Desarrollo metodológico del clúster

El análisis de clúster agrupa observaciones similares entre sí y diferentes de las demás. En este laboratorio se recomienda aplicar el clúster sobre los scores del PCA, no directamente sobre todas las variables originales, porque los scores reducen redundancia, estabilizan distancias y facilitan la interpretación.

La segmentación debe responder a una pregunta de gestión: por ejemplo, identificar sitios con alta presión y baja capacidad institucional, comunidades con alta vulnerabilidad ecológica y baja infraestructura, o territorios con baja presión y alta conservación que requieren prevención más que restauración.

5.1 Decisiones clave

DecisiónRecomendaciónJustificación

EscalaTrabajar con datos estandarizados antes del PCA.Las distancias son sensibles a la escala.
MétodoK-means para segmentación operativa; jerárquico para explorar estructura.K-means es eficiente; jerárquico ayuda a visualizar agrupamientos.
Número de gruposComparar k = 2 a 7 o un rango justificado.Evita aceptar un único resultado sin contraste.
ValidaciónCombinar silhouette, codo, tamaño de grupos y sentido sustantivo.Un k con buen indicador puede ser poco útil para política pública.
Regla de interpretación: no etiquete los clústeres como “buenos” o “malos” sin analizar sus perfiles. Use nombres descriptivos: “alta presión / baja capacidad”, “presión media / gobernanza fuerte”, “baja presión / alta biodiversidad”.

7. Errores frecuentes que deben evitarse

  1. Ejecutar PCA sin pregunta: la técnica debe responder a un problema de investigación o gestión, no sustituirlo.
  2. No estandarizar: provoca que variables con mayor rango dominen la solución.
  3. Confundir correlación con causalidad: PCA y clúster describen patrones; no prueban causalidad.
  4. Sobreinterpretar componentes débiles: si las cargas no tienen estructura clara, la etiqueta debe ser prudente.
  5. Elegir k solo por software: el número de clústeres debe ser estadísticamente razonable y sustantivamente útil.
  6. Ignorar el territorio: los resultados deben contrastarse con mapas, trabajo de campo, expertos o actores locales.
  7. Usar clústeres como ranking: un clúster no es automáticamente mejor o peor; representa un perfil.

9. Código base para Google Colab

El siguiente código genera un ejemplo reproducible con datos sintéticos. La estructura simula relaciones plausibles entre presión antrópica, calidad ambiental y capacidad de gestión. En una aplicación real, sustituya esta base por datos observados y mantenga la misma trazabilidad metodológica.

# Laboratorio PCA + Clúster aplicado a gestión marina costera
# Versión metodológicamente reforzada para Google Colab

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt

from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
from sklearn.cluster import KMeans, AgglomerativeClustering
from sklearn.metrics import silhouette_score
from scipy.cluster.hierarchy import dendrogram, linkage

# ==========================================================
# 1. Crear datos sintéticos con estructura correlacional
# ==========================================================
# En una investigación real, esta sección se sustituye por:
# df = pd.read_csv("archivo_real.csv")

np.random.seed(123)
n = 180

# Variables base. Se simulan como indicadores de sitios, comunidades o unidades costeras.
df = pd.DataFrame({
    'presion_turistica': np.random.normal(60, 15, n),
    'esfuerzo_pesquero': np.random.normal(55, 18, n),
    'calidad_agua': np.random.normal(70, 12, n),
    'gobernanza_local': np.random.normal(50, 14, n),
    'cumplimiento_normativo': np.random.normal(58, 16, n),
    'conflictividad': np.random.normal(45, 17, n),
    'infraestructura': np.random.normal(62, 15, n),
    'biodiversidad': np.random.normal(65, 13, n)
})

# Indicadores compuestos simulados.
# degradacion aumenta con presión turística y esfuerzo pesquero,
# y disminuye cuando mejora la calidad del agua.
df['degradacion'] = (
    0.55*df['presion_turistica'] +
    0.45*df['esfuerzo_pesquero'] -
    0.35*df['calidad_agua'] +
    np.random.normal(0, 8, n)
)

# capacidad_gestion aumenta con gobernanza, cumplimiento e infraestructura.
df['capacidad_gestion'] = (
    0.50*df['gobernanza_local'] +
    0.45*df['cumplimiento_normativo'] +
    0.30*df['infraestructura'] +
    np.random.normal(0, 7, n)
)

variables = df.columns.tolist()

# ==========================================================
# 2. Revisión inicial de calidad de datos
# ==========================================================
print("Dimensiones de la base:", df.shape)
print("\nValores perdidos por variable:")
print(df[variables].isna().sum())

print("\nResumen descriptivo:")
print(df[variables].describe().round(2))

# Matriz de correlación para verificar si PCA tiene sentido.
correlaciones = df[variables].corr()
print("\nMatriz de correlaciones:")
print(correlaciones.round(2))

# ==========================================================
# 3. Estandarización
# ==========================================================
# PCA y clúster son sensibles a la escala. Por eso se estandariza.
scaler = StandardScaler()
X = scaler.fit_transform(df[variables])

# ==========================================================
# 4. PCA exploratorio
# ==========================================================
pca = PCA()
pca.fit(X)

varianza = pca.explained_variance_ratio_
var_acum = np.cumsum(varianza)

tabla_varianza = pd.DataFrame({
    'componente': [f'PC{i+1}' for i in range(len(varianza))],
    'varianza_explicada': varianza,
    'varianza_acumulada': var_acum
})
print("\nVarianza explicada por componente:")
print(tabla_varianza.round(3))

# Criterio pedagógico: retener componentes hasta explicar al menos 75% de varianza.
# Este criterio debe contrastarse con scree plot e interpretabilidad.
n_comp = np.argmax(var_acum >= 0.75) + 1
print("\nComponentes retenidos por criterio de 75% de varianza acumulada:", n_comp)

# Scree plot
plt.figure(figsize=(8, 5))
plt.plot(range(1, len(varianza)+1), varianza, marker='o')
plt.xlabel('Componente principal')
plt.ylabel('Varianza explicada')
plt.title('Scree plot del PCA')
plt.grid(True, alpha=0.3)
plt.show()

# PCA final con componentes retenidos
pca_final = PCA(n_components=n_comp)
scores = pca_final.fit_transform(X)

# Cargas: indican qué variables definen cada componente.
cargas = pd.DataFrame(
    pca_final.components_.T,
    index=variables,
    columns=[f'PC{i+1}' for i in range(n_comp)]
)
print("\nCargas del PCA:")
print(cargas.round(3))

# Scores por observación
scores_df = pd.DataFrame(scores, columns=[f'PC{i+1}' for i in range(n_comp)])

# ==========================================================
# 5. Selección del número de clústeres
# ==========================================================
# Se evalúa k entre 2 y 7. Puede ampliarse según el problema.
silhouette = {}
inercia = {}

for k in range(2, 8):
    modelo = KMeans(n_clusters=k, n_init=50, random_state=123)
    etiquetas = modelo.fit_predict(scores_df)
    silhouette[k] = silhouette_score(scores_df, etiquetas)
    inercia[k] = modelo.inertia_

print("\nSilhouette por número de clústeres:")
print(pd.Series(silhouette).round(3))

# Gráfico de silhouette
plt.figure(figsize=(8, 5))
plt.plot(list(silhouette.keys()), list(silhouette.values()), marker='o')
plt.xlabel('Número de clústeres k')
plt.ylabel('Silhouette promedio')
plt.title('Selección de k por silhouette')
plt.grid(True, alpha=0.3)
plt.show()

# Gráfico del codo
plt.figure(figsize=(8, 5))
plt.plot(list(inercia.keys()), list(inercia.values()), marker='o')
plt.xlabel('Número de clústeres k')
plt.ylabel('Inercia intra-clúster')
plt.title('Método del codo')
plt.grid(True, alpha=0.3)
plt.show()

# Selección automática inicial: k con mayor silhouette.
# En el informe, esta selección debe discutirse con sentido sustantivo.
k_opt = max(silhouette, key=silhouette.get)
print("\nK óptimo inicial por silhouette:", k_opt)

# ==========================================================
# 6. Modelo final de clúster
# ==========================================================
modelo_final = KMeans(n_clusters=k_opt, n_init=100, random_state=123)
df['cluster'] = modelo_final.fit_predict(scores_df)

# Perfil de clústeres usando variables originales.
perfil = df.groupby('cluster')[variables].mean().round(2)
tamano = df['cluster'].value_counts().sort_index().rename('n')
porcentaje = (df['cluster'].value_counts(normalize=True).sort_index()*100).round(1).rename('%')

print("\nTamaño de clústeres:")
print(pd.concat([tamano, porcentaje], axis=1))

print("\nPerfil promedio de variables originales por clúster:")
print(perfil)

# ==========================================================
# 7. Visualización de clústeres sobre los dos primeros componentes
# ==========================================================
if n_comp >= 2:
    plt.figure(figsize=(8, 6))
    plt.scatter(scores_df['PC1'], scores_df['PC2'], c=df['cluster'], alpha=0.75)
    plt.xlabel('PC1')
    plt.ylabel('PC2')
    plt.title('Clústeres proyectados sobre los dos primeros componentes')
    plt.grid(True, alpha=0.3)
    plt.show()

# ==========================================================
# 8. Clúster jerárquico como contraste exploratorio
# ==========================================================
Z = linkage(scores_df, method='ward')
plt.figure(figsize=(10, 5))
dendrogram(Z, truncate_mode='level', p=5)
plt.title('Dendrograma jerárquico sobre scores PCA')
plt.xlabel('Observaciones agrupadas')
plt.ylabel('Distancia')
plt.show()

# ==========================================================
# 9. Guía mínima de interpretación automática
# ==========================================================
print("\nGuía de interpretación:")
print("1. Revise qué variables tienen mayores cargas en cada componente.")
print("2. Asigne nombres sustantivos a los componentes solo si las cargas son coherentes.")
print("3. Interprete cada clúster con las medias de variables originales, no solo con los scores.")
print("4. Compare la solución estadística con conocimiento territorial y objetivos de gestión.")
print("5. Reporte limitaciones: datos sintéticos, supuestos de escala, sensibilidad a outliers y carácter exploratorio.")
