2. Supuestos de simulación

Esta sección no usa datos reales: define las reglas para crear una base sintética parecida al problema del estudiante. La idea es que el estudiante comprenda qué está suponiendo antes de correr una regresión. Cada parámetro controla una parte distinta del proceso de simulación.

Lectura pedagógica: primero se genera una población artificial con las variables X. Luego se construye Y como una combinación de esas X, un punto de partida llamado intercepto y un componente de error aleatorio. Después se ajusta el modelo y se revisa si los supuestos se cumplen o fallan.
Cantidad de filas que tendrá la base sintética. Un N pequeño sirve para ver inestabilidad; un N mayor permite patrones más claros.
Valor esperado de Y cuando las X valen cero. Funciona como punto de partida del modelo simulado.
Nivel de ruido no explicado por las X. Si aumenta, el modelo tendrá menor ajuste y residuales más dispersos.
Número que permite repetir exactamente la misma simulación. Si cambia, los datos generados también cambian.
Permite provocar condiciones específicas para practicar diagnóstico: outliers, heterocedasticidad, no normalidad o multicolinealidad.
Define si la salida se genera en R, Python/Colab o ambos. Para esta sesión puede priorizar R si el laboratorio se trabajará con lm() y plot(modelo).
